# Visual System v1

## Главный визуальный элемент

Главный визуальный элемент фабрикод — **Параметрическая рамка**.

Это не декоративная рамка и не чертеж. Это способ показать, что изделие описано цифровыми параметрами и правилами, а не ручной работой отдельного специалиста.

Смысл:

> Изделие находится внутри системы параметров. Изменение параметра запускает пересчет цены, спецификации и производственных файлов.

## Конструкция

Параметрическая рамка состоит из пяти слоев:

1. **Контур объекта** — тонкая рамка вокруг изделия, mockup или ключевого числа.
2. **Угловые маркеры** — короткие линии в углах, создающие ощущение выбора и измерения.
3. **Точки параметров** — маленькие узлы, обозначающие управляемые значения.
4. **Связи правил** — короткие линии от параметра к результату.
5. **Выходные метки** — компактные labels: `Цена`, `SPEC`, `DXF`, `NC`, `CRM`, `1C`.

## Базовая схема

```text
┌ ─ ─ ─ ─ ─ ─ ─ ─ ┐
  [ объект / mockup ]  • параметр
│                   ─── Цена
  • правило           ─── SPEC
└ ─ ─ ─ ─ ─ ─ ─ ─ ┘
```

В реальном дизайне углы не должны быть полным чертежным прямоугольником. Лучше использовать разорванную рамку: 4 угла + 1-3 активные связи.

## Цвета

### Основные

- фон: `#f7f8fa`;
- поверхность: `#ffffff`;
- текст: `#1f2329`;
- вторичный текст: `#7d8794`;
- активный параметр: `#b42335`;
- темная связь или темный mockup: `#15181d`;
- разделители: `#e5e8ec`.

### Акцентный градиент

```css
linear-gradient(135deg, #b42335 0%, #5c1020 100%)
```

Применять только к:

- CTA;
- активной линии;
- активной точке;
- ключевому числу;
- небольшому статусу.

Не применять к большим фоновым заливкам без необходимости.

## Размеры

### Desktop

| Элемент | Размер |
|---|---:|
| Толщина основной линии | `1px` |
| Толщина активной линии | `2px` |
| Длина углового маркера | `28-48px` |
| Точка параметра | `7-10px` |
| Активная точка | `10-14px` |
| Радиус маленькой label-метки | `8-10px` |
| Отступ рамки от объекта | `20-40px` |

### Mobile

| Элемент | Размер |
|---|---:|
| Толщина линии | `1px` |
| Длина углового маркера | `18-28px` |
| Точка параметра | `6-8px` |
| Отступ рамки от объекта | `12-20px` |

На мобильных лучше уменьшать количество связей, а не масштабировать всю сложную схему.

## Отступы

Правило внутреннего воздуха:

- маленькая карточка: рамка отступает от контента на `16-20px`;
- hero/mockup: `28-40px`;
- презентационный слайд: `40-64px`;
- выставочный стенд: масштабировать пропорционально, но оставлять широкие поля.

Не размещать текст вплотную к линиям рамки. Минимальный отступ текста от линии: `12px`.

## Варианты построения

### 1. Рамка вокруг продукта

Используется в hero и продуктовых блоках.

Состав:

- темный mockup интерфейса;
- тонкая светлая рамка;
- 1 активная бордовая точка;
- 1-2 labels с результатом.

Пример:

```text
[Параметры шкафа] -> [148 500 ₽]
                  -> [DXF / NC / SPEC]
```

### 2. Рамка вокруг числа

Используется в экономике и отчетах.

Состав:

- крупное число;
- короткие угловые маркеры;
- маленькая подпись;
- один бордовый акцент.

Пример:

```text
4 800 000 ₽
потери в год
```

### 3. Рамка процесса

Используется для схем.

Состав:

- 4-5 узлов;
- короткие связи;
- активный текущий шаг;
- выходные метки.

Пример:

```text
Параметры -> Правила -> Цена -> SPEC -> Производство
```

### 4. Фоновый паттерн

Используется в презентациях, PDF и соцсетях.

Состав:

- очень тонкие линии `rgba(31, 35, 41, 0.05)`;
- редкие точки;
- без большого количества labels;
- акцент максимум в одном месте.

### 5. Интерфейсный маркер

Используется внутри продукта.

Состав:

- маленькая активная точка;
- линия к выбранному параметру;
- value-label;
- статус пересчета.

## Правила использования

### Делать

- использовать рамку для объяснения параметров и правил;
- оставлять много воздуха;
- выделять только один активный параметр;
- сочетать со светлыми карточками и темными product mockup;
- держать линии тонкими;
- использовать бордовый как смысловой акцент.

### Не делать

- не превращать рамку в чертежную сетку на весь экран;
- не добавлять шестеренки, ключи, трубы и заводские силуэты;
- не использовать слишком много узлов;
- не делать все линии бордовыми;
- не накладывать паттерн на длинный текст;
- не делать фон темным на всей странице.

## Типографика рядом с элементом

Labels:

- размер: `10-12px`;
- вес: `600-650`;
- uppercase допустим только для технических меток: `DXF`, `NC`, `SPEC`;
- цвет: `#7d8794`, активный label `#b42335`.

Заголовки внутри рамки:

- вес: `600-700`;
- без чрезмерного `800`;
- line-height: `1.1-1.25`.

Описание:

- вес: `400`;
- цвет: `#7d8794`;
- line-height: `1.55-1.7`.

## Анимация

Анимация должна показывать процесс, а не украшать.

Рекомендуемая последовательность:

1. появляется объект;
2. появляются точки параметров;
3. активируется одна точка;
4. линия идет к цене;
5. цена пересчитывается;
6. появляются выходные метки `SPEC`, `DXF`, `NC`.

Параметры:

- duration: `180-420ms` для UI;
- duration: `600-1200ms` для hero-анимации;
- easing: `cubic-bezier(.2,.8,.2,1)`;
- без резких bounce-эффектов.

## Примеры применения

### Сайт

Hero:

- темный mockup конфигуратора справа;
- вокруг mockup Параметрическая рамка;
- активная точка у параметра;
- бордовая связь к цене и файлам.

Карточка отрасли:

- белая карточка;
- номер `01`;
- тонкий угловой маркер;
- без иллюстративной перегрузки.

Калькулятор:

- крупное бордовое число;
- вокруг результата легкие угловые маркеры;
- вторичные значения спокойные серые.

### Презентации

Обложка:

- светлый фон;
- крупный заголовок;
- справа или снизу крупная Параметрическая рамка;
- 1 активная бордовая точка;
- без фонового шума.

Слайд процесса:

- горизонтальная схема;
- активный шаг бордовый;
- остальные шаги графитовые/серые;
- рамка объединяет схему как единую систему.

### PDF

Кейс:

- рамка вокруг ключевого результата;
- диаграммы с бордовым главным значением;
- таблицы с мягкими разделителями;
- технические метки `SPEC`, `DXF`, `NC` как маленькие labels.

### Соцсети

Пост:

- один тезис;
- одно число;
- Параметрическая рамка вокруг числа или mockup;
- минимум текста.

### Интерфейс продукта

Конфигуратор:

- активный параметр отмечен бордовой точкой;
- зависимые значения подсвечиваются мягко;
- технические выходы группируются в labels;
- темная зона используется только для preview или технического mockup.

## 5 концепций фирменного элемента

### 1. Параметрическая рамка

Идея: изделие или интерфейс помещается в систему угловых маркеров, точек параметров и связей.

Плюсы: универсальна, связана с продуктом, масштабируется, не выглядит индустриальным клише.

Минусы: требует дисциплины и умеренности.

Применение: главный элемент бренда.

### 2. Размерные линии

Идея: тонкие измерительные линии показывают точность и параметрическое описание.

Плюсы: понятная инженерная метафора, хорошо работает с изделием.

Минусы: может выглядеть слишком чертежно.

Применение: вторичный элемент для продуктовых визуалов.

### 3. Конфигурационная сетка

Идея: сетка модулей, где активные ячейки обозначают параметры и состояния.

Плюсы: SaaS-ощущение, удобно для фонов.

Минусы: менее уникальна без других элементов.

Применение: фоновые паттерны, презентации, интерфейсные состояния.

### 4. Fabricode Frame

Идея: фирменный контейнер с асимметричными углами и одной активной точкой.

Плюсы: простой узнаваемый прием, хорошо держит контент.

Минусы: больше декоративный, чем смысловой.

Применение: кейсы, баннеры, карточки.

### 5. Алгоритмический паттерн

Идея: сеть зависимостей между параметрами, правилами и результатами.

Плюсы: хорошо раскрывает автоматизацию.

Минусы: может стать generic tech-графикой.

Применение: motion, схемы, technical slides.

## Выбранная концепция

Основная концепция: **Параметрическая рамка**.

Она лучше всего подходит фабрикод, потому что одновременно говорит о трех вещах:

- изделие описывается параметрами;
- параметры связаны правилами;
- результат уходит в производство как цифровые данные.

Это не мебельная, не заводская и не абстрактная IT-метафора. Это визуальный перевод сути продукта.

## Контроль качества

Перед публикацией любого материала проверить:

- бренд написан как `фабрикод`;
- фон преимущественно светлый;
- бордовый акцент используется точечно;
- нет шестеренок и индустриальных клише;
- Параметрическая рамка объясняет смысл, а не просто украшает;
- текст не перегружен жирными весами;
- один макет содержит одну главную мысль;
- визуал связан с параметрами, правилами или производственными файлами.
